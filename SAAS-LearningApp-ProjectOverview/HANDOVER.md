## How it works (important)

- **Single source of truth for navigation.** The top nav, mobile drawer, and footer are all generated by `js/site.js` from the `PAGES` and `PHASES` arrays. Each page only needs `<div id="site-nav"></div>`, `<div id="site-footer"></div>`, `<body data-page="…">`, and `<script src="js/site.js"></script>`. **To add, remove, or rename a page, edit `js/site.js` only** — do not hand-edit nav markup per page.
- **Theme.** Light by default with a dark toggle (persisted in `localStorage` as `ll-theme`). A tiny inline script in each `<head>` sets the theme before paint to avoid a flash; the toggle button lives in the nav.
- **Phase numbering (canonical):** 1 MVP · 2 Auth & accounts · 3 Expo/Native · 4 Lumi Plus · 5 Lumi Schools. Keep this consistent everywhere.

## Tech

- Static HTML + CSS + vanilla JS, no build step.
- Fonts: Baloo 2 + Nunito. No external JS libraries (Mermaid was removed in favour of a CSS timeline).
- Local preview: `python -m http.server 8081` (8080 is used by lekkeleer-lees).
- Deploy: GitHub Pages (root of branch).

---

## When ROADMAP.md changes

1. Open `lekkeleer-lees/docs/ROADMAP.md`.
2. Update `roadmap.html` (timeline + per-phase tables) and the matching `phase-N.html`.
3. If monetization changes, update `monetization.html` (and the reference tables in `roadmap.html`).
